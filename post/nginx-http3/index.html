在NGINX上启用HTTP/3的注意事项#

HTTP/3简介#

HTTP/3是HTTP（超文本传输协议）协议的第三代标准，它标志着HTTP协议自HTTP/2以来的首次重大升级。HTTP/3引入了基于UDP（用户数据报协议）的新传输层协议QUIC，旨在解决TCP在网络性能上的局限性，特别是针对移动设备频繁切换网络环境的需求。QUIC协议不仅继承了TCP的可靠性特性，还引入了UDP的灵活性和速度优势，从而克服了TCP的队头阻塞（Head-of-Line Blocking）问题，提高了连接建立速度，并支持零往返时间（0-RTT）的数据传输。

相对于HTTP/2协议，HTTP/3协议带来了重大的升级。它放弃了以往HTTP协议采用的TCP数据传输协议，转而采用了基于UDP数据传输协议的QUIC。QUIC协议的设计初衷是为了克服TCP在面对数据包丢失或网络延迟时导致的整体性能下降问题，即所谓的“队头阻塞”。通过在UDP基础上重新实现TCP的可靠性机制，QUIC允许独立的数据流在遭遇网络拥塞时仅阻塞受影响的那一路数据，而非整个连接，这显著缓解了TCP协议导致的“队头阻塞”问题。

此外，HTTP/3实现了全链路默认加密，这一新特性增强了数据传输的安全性。在传统的TCP连接中，加密与传输层是分离的，这意味着TCP连接可以携带加密或非加密的数据，而握手过程与加密过程是分开的。在QUIC中，加密是在传输层进行的，这意味着应用层的数据始终处于加密状态，这不仅提高了安全性，还减少了握手过程中的延迟。

QUIC协议的优势#

QUIC协议通过将HTTP/2的一些概念引入到传输层，解决了TCP的队头阻塞问题。这意味着当某个数据包在网络中丢失或严重延迟时，QUIC能够区分不同流的数据，并且不会因为一个流的数据包丢失而阻碍其他流的数据传输。这种机制使得QUIC能够更好地支持移动设备频繁切换网络的场景，因为QUIC可以在网络切换时不中断连接。

性能提升#

HTTP/3与QUIC的结合带来了若干性能上的改进。首先，QUIC减少了建立连接所需的时间，这得益于它将加密和传输握手整合为一次握手，从而减少了延迟。其次，QUIC支持零往返时间（0-RTT）连接建立，对于已经建立了信任关系的客户端和服务器之间，客户端可以在首次握手之前就开始发送数据，大大加快了数据传输的速度。最后，QUIC对拥塞控制的改进使得它在网络拥塞时能够更快地调整传输速率，从而优化了网络资源的使用。

综上所述，HTTP/3与QUIC的组合通过解决TCP的固有问题，为Web性能带来了实质性的改善。尽管这种改善在大多数情况下可能不如预期的那么显著，尤其是在网络状况良好的环境中，但对于那些面临较高丢包率或网络不稳定情况的应用来说，这些改进可以带来显著的好处。

NGINX上安装HTTP/3模块#

在 NGINX 官方文档上已经声明，在 NGINX-1.25.0 正式引入 HTTP/3 与 QUIC 并作为实验性的一部分受到支持。由于 Openssl 并不支持早期数据，所以在安装时要指定支持QUIC 协议的SSL数据库。例如例如 BoringSSL、LibreSSL或 QuicTLS。

首先，我们需要编译安装NGINX软件，下面，我将简单说明在下载安装之前的准备工作。

该示例的操作系统为 Ubuntu 22.04.4 LTS。

#更新软件源
sudo apt-get update
#更新所有软件
sudo apt-get upgrade
#安装必要软件
sudo apt-get install wegt vim build-essential libpcre3 libpcre3-dev zlib1g zlib1g-dev libssl-dev libgd-dev libxml2 libxml2-dev uuid-dev -y
#创建工作目录
cd / && sudo mkdir work && cd work
#下载大于等于1.25.0版本的NGINX，下面以1.26.2为例
#NGINX下载页面 https://nginx.org/en/download.html
wget https://nginx.org/download/nginx-1.26.2.tar.gz
#下载支持QUIC的SSL数据库
#以LibreSSL为例，同时以注释的方式提供其他方式下载指令。
wget https://ftp.openbsd.org/pub/OpenBSD/LibreSSL/libressl-3.9.2.tar.gz
# QUIC SSL
#wget https://github.com/quictls/openssl/archive/refs/tags/openssl-3.1.6-quic1.tar.gz
#BoringSSL，采用了git方式来获取源码
#git clone https://boringssl.googlesource.com/boringssl

#解压缩打包好的软件源码
tar -zxvf nginx-1.26.2.tar.gz && tar -zxvf libressl-3.9.2.tar.gz
#查看工作目录下已经解压好的目录
ls -la
#输出结果
root@:Ubuntu/work# ls -la
total 5484
drwxr-xr-x  4 root root     4096 Sep  3 13:36 .
drwxr-xr-x 20 root root     4096 Sep  3 13:25 ..
drwxr-xr-x 11 root root     4096 Sep  3 13:36 libressl-3.9.2
-rw-r--r--  1 root root  4353829 May 12 18:32 libressl-3.9.2.tar.gz
drwxr-xr-x  8  502 staff    4096 Aug 13 00:39 nginx-1.26.2
-rw-r--r--  1 root root  1244789 Aug 13 00:39 nginx-1.26.2.tar.gz

自此，所有的准备工作就绪。下面，来实现编译时支持QUIC的构建工作。

#切换到NGINX目录下
cd nginx-1.26.2
#构建编译指令
./configure 
--with-http_v3_module  \
--with-http_ssl_module \
--with-openssl=/work/libressl-3.9.2

这里需要注意的是，由于默认下NGINX不会构建HTTP3和SSL的模块，所以在编译时需要我们自己指定安装模块，同时为了避免NGINX在安装时候使用系统自带的OpenSSL，同时也需要我们指定对应的支持QUIC的SSL数据库源码。

对于下载的SSL数据库可以不进行编译安装，在NGINX编译安装时候，由于指定了SSL数据库的源码，所以在NGINX编译安装的时候会同时编译SSL数据库。

当NGINX指定好编译参数后就可以正式进入安装阶段。

make && make install

当我们安装完成NGINX后，MAKE套件会自动清理安装产生的安装缓存。

这时候，我们来验证NGINX是安装完成。

由于我们在安装的时候并没有指定NGINX安装目录，此时，NGINX安装目录是默认的/usr/local/nginx。并且我们没有创建PATH系统变量，所以我们要去到NGINX安装目录下的sbin目录下来执行NGINX有关的命令。

cd /usr/local/nginx/sbin
#查看NGINX版本和详细信息
./nginx -V
#输出结果是
nginx version: nginx/1.26.2
built with LibreSSL 3.9.2
TLS SNI support enabled
configure arguments: --with-http_v3_module --with-http_ssl_module --with-openssl=/root/server/libressl-3.9.2

此时，NGINX向我们表示，NGINX已经成功加载了HTTP3和SSL模块，并且使用了我们指定的SSL数据库。

在NGINX的配置文件开启HTTP3及QUIC新特性#

注意，由于HTTP3和QUIC的默认加密特性，需要在NGINX上绑定域名并拥有该域名的签发的SSL证书才可以开启。

这里，我们需要找到NGINX配置文件，来开启HTTP3和QUIC新特性。

假如，您并不知道你的NGINX配置文件在哪里，你可以使用find命令来查找该文件。

find / -name nginx.conf

这样，系统会从系统根目录查找名为nginx.conf文件。

接下来，我们来配置该文件，用来开启HTTP3与QUIC特性。

假设我们现在处于NGINX配置文件同目录下。

#备份nginx.conf
cp nginx.conf nginx.cong.backup
#编辑nginx.conf
vim nginx.conf

此时，nginx.conf示例样子是这样的

worker_processes  1;
events {
    worker_connections  1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
...

    server {
        listen       80;
        server_name  localhost;

...

        }
    }
}

下面，我们来修改这份配置文件，来使NGINX支持HTTP3和QUIC。

对于开启HTTP3，首先要现在HTTP块设置开启HTTP3的指令，开启HTTP3的协议协商并控制HTTP3最大的请求数。

对于启用QUIC，同样的也需要在HTTP块中开启。

http{
#HTTP3开启代码块
http3 						 on;
http3_hp 					 on;
 #对于连接数控制，可以自定义
http3_stream-buffer_size 	  64k;
#QUIC开启代码块
quic_bpf					on;
quic_gso					on;
...
}

同时，我们需要告知浏览器我们使用了HTTP3协议和QUIC协议，我们需要在server块中添加一个响应标头

server{
...
 add_header Alt-Svc  'h3=":443"; ma=86400';
...
}

这样，在浏览器访问我们的网页的时候，会在浏览器控制台中对响应的资源添加了名为h3的响应头。

随后，我们需要在配置文件里面修改好我们已经配置好的域名和对应的SSL证书。

自此，我们修改好了对于网页的HTTP3的特性配置。

现在，我们来检测我们修改的配置文件以及对应的文件配置是否正确。

./nginx -t
#如果没有任何问题，这会返回如下结果
nginx: the configuration file /***/nginx.conf syntax is ok

如果没有任何问题，我们可以执行重启了。

./nginx -s reopen

在没有任何问题情况下，NGINX会自动完启动。

此时，我们访问对应的域名，打开安全选项卡后，我们会发现，网站正在使用QUIC来传输数据。

后记#

  • 由于HTTP3和QUIC特性，我们在配置这些特性时候，需要一个域名和对应签发的SSL证书。
  • 对于QUIC来说，由于使用了UDP协议，所以我们需要同时打开服务器的UDP访问端口。
  • 对于HTTP2和HTTP3协议，NGINX支持端口复用和TCP复用，所以我们在监听端口时候可以直接配置一个端口并表明该端口被复用。
